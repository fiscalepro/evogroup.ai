## Находки

### [V-001] HIGH: Ключи KGZ могут не совпадать с RU после добавления новых файлов
**Сценарий:** Builder утверждает что ключи kgz = ключи ru.
**Атака:** aichatbot.json в RU имеет синтаксическую ошибку (строка 19 — незакрытая фигурная скобка в quick_replies). KGZ версия написана с исправленным JSON. Структуры могут расходиться.
**Последствие:** Фронтенд запросит ключ который есть в ru но отсутствует в kgz → покажет пустоту или ключ.
**Рекомендация:** Запустить автоматическое сравнение ключей ru vs kgz для всех 17 файлов.

### [V-002] MEDIUM: aicalculator.json не был затронут исправлениями
**Сценарий:** Все файлы проверены на грамматику.
**Атака:** aicalculator.json (227 строк) существовал до правок и не был включён в список исправленных файлов. Содержит ли он те же ошибки (корпоративдүү, проактивдүү, и т.д.)?
**Последствие:** Возможны оставшиеся грамматические ошибки в большом файле.
**Рекомендация:** Проверить aicalculator.json на те же паттерны.

### [V-003] MEDIUM: src/locales/kgz/ содержит дублирующие файлы
**Сценарий:** Исправлены файлы в public/locales/kgz/.
**Атака:** В проекте есть вторая копия локалей в src/locales/kgz/ (9 файлов). Они НЕ были исправлены. Если фронтенд импортирует из src/locales/ — правки не подействуют.
**Последствие:** Часть сайта может показывать старые/ошибочные переводы.
**Рекомендация:** Проверить откуда фронтенд берёт переводы, исправить src/locales/kgz/ если нужно.

### [V-004] LOW: locale может быть undefined в API
**Сценарий:** API деструктурирует locale из body.
**Атака:** Если старый клиент (кэш) не шлёт locale, значение будет undefined → langMap[undefined] = undefined → responseLang = 'Russian' (fallback). Это не баг, но неявное поведение.
**Последствие:** Нет — fallback на русский корректен.

### [V-005] LOW: Нет автотестов на локализацию
**Сценарий:** Все проверки ручные.
**Атака:** Регрессия неизбежна. Следующий разработчик может добавить ключ в ru и забыть kgz.
**Рекомендация:** Добавить CI проверку: ключи ru === ключи kgz для каждого файла.
